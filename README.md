## 1. Что такое база данных?

**База данных (БД)** — это организованная коллекция структурированных данных, хранящихся и управляемых на компьютере. База данных позволяет удобно хранить, искать, сортировать и обрабатывать большие объёмы информации.

### Основные характеристики БД:

- **Структурированность** — данные хранятся в определённом формате (таблицы, строки, столбцы)
- **Связанность** — таблицы могут быть связаны между собой
- **Целостность** — данные защищены от дублирования и противоречий
- **Доступность** — данные могут извлекаться по запросу

### Примеры баз данных в жизни:

| Сфера | Пример БД |
|-------|-----------|
| Медицина | Картотека пациентов больницы |
| Образование | Журнал успеваемости учеников |
| Торговля | Каталог товаров интернет-магазина |
| Банки | Счета и транзакции клиентов |
| Библиотека | Каталог книг и читателей |

## 2. Что такое СУБД?

**СУБД (Система управления базами данных)** — это программное обеспечение, которое обеспечивает создание, хранение, редактирование и управление базами данных.

СУБД выступает посредником между пользователем и данными: пользователь отправляет запрос → СУБД обрабатывает его → возвращает результат.

### Функции СУБД:

- ✅ Создание и изменение структуры БД
- ✅ Добавление, редактирование и удаление данных
- ✅ Поиск и выборка данных по условиям
- ✅ Контроль доступа и защита данных
- ✅ Обеспечение целостности данных
- ✅ Резервное копирование и восстановление

### Популярные СУБД:

| СУБД | Тип | Применение |
|------|-----|------------|
| **MS Access** | Настольная | Малый и средний бизнес |
| **MySQL** | Серверная | Веб-приложения |
| **PostgreSQL** | Серверная | Корпоративные системы |
| **Oracle** | Серверная | Крупные предприятия |
| **SQLite** | Встраиваемая | Мобильные приложения |

## 3. Назначение MS Access

**Microsoft Access** — это реляционная СУБД, входящая в состав пакета Microsoft Office. Разработана компанией Microsoft и предназначена для создания и управления базами данных на персональных компьютерах.

### Основное назначение:

> MS Access позволяет создавать полноценные базы данных без глубоких знаний программирования, используя удобный графический интерфейс.

MS Access предназначен для:

- 📁 **Хранения данных** — организация информации в таблицах
- 🔍 **Поиска и выборки** — создание запросов для получения нужных данных
- 📋 **Ввода данных** — удобные формы для работы с информацией
- 📊 **Формирования отчётов** — создание печатных документов с итогами
- 🔗 **Связи таблиц** — построение реляционных структур данных

### Версии MS Access:

```
Access 2.0 → Access 97 → Access 2003 → Access 2007 → Access 2010
→ Access 2013 → Access 2016 → Access 2019 → Access 2021 (Microsoft 365)
```

## 4. Основные объекты MS Access

MS Access содержит 4 основных объекта для работы с данными:

---

### 🗃️ Таблицы (Tables)

**Таблица** — основной объект базы данных. Именно в таблицах хранятся все данные. Таблица состоит из **строк (записей)** и **столбцов (полей)**.

```
Таблица "Студенты":
┌────┬──────────────┬─────────┬───────────┬──────────┐
│ ID │ Фамилия      │ Имя     │ Группа    │ Оценка   │
├────┼──────────────┼─────────┼───────────┼──────────┤
│  1 │ Иванов       │ Алексей │ ИС-21     │   4.5    │
│  2 │ Петрова      │ Мария   │ ИС-21     │   5.0    │
│  3 │ Сидоров      │ Дмитрий │ ПО-22     │   3.8    │
└────┴──────────────┴─────────┴───────────┴──────────┘
```

**Особенности таблиц:**
- Каждая таблица хранит данные об одном типе объектов
- Каждый столбец имеет определённый тип данных
- Каждая строка — отдельная запись (уникальная единица информации)
- Обязательно наличие первичного ключа

---

### 🔎 Запросы (Queries)

**Запрос** — инструмент для выборки, фильтрации и обработки данных из одной или нескольких таблиц. Запросы создаются с помощью языка **SQL** или визуального конструктора.

**Типы запросов:**

| Тип | Назначение |
|-----|-----------|
| **Запрос на выборку** | Извлекает данные по условию |
| **Запрос на добавление** | Добавляет записи в таблицу |
| **Запрос на обновление** | Изменяет существующие записи |
| **Запрос на удаление** | Удаляет записи по условию |
| **Перекрёстный запрос** | Создаёт сводные таблицы |

**Пример SQL-запроса:**
```sql
SELECT Фамилия, Имя, Оценка
FROM Студенты
WHERE Оценка >= 4.0
ORDER BY Оценка DESC;
```

---

### 📝 Формы (Forms)

**Форма** — графический интерфейс для удобного ввода, просмотра и редактирования данных в таблице. Форма делает работу с базой данных удобной даже для неподготовленного пользователя.

**Возможности форм:**
- Отображение одной записи на экране
- Добавление кнопок и элементов управления
- Проверка введённых данных
- Создание главной/подчинённой формы (для связанных таблиц)
- Настройка внешнего вида (цвета, шрифты, логотипы)

**Элементы формы:**
```
[Поле ввода]  [Список выбора ▼]  [Флажок ☑]
[Кнопка: Сохранить]  [Кнопка: Отмена]
[Надпись]  [Рисунок]  [Вкладки]
```

---

### 📄 Отчёты (Reports)

**Отчёт** — объект для вывода данных на печать в удобном, отформатированном виде. Отчёты позволяют группировать данные, подводить итоги и создавать профессиональные документы.

**Возможности отчётов:**
- Группировка и сортировка данных
- Подсчёт итогов (сумма, среднее, количество)
- Создание диаграмм и графиков
- Добавление верхних/нижних колонтитулов
- Экспорт в PDF, Word, Excel

**Структура отчёта:**
```
┌─────────────────────────────────────┐
│          ЗАГОЛОВОК ОТЧЁТА           │
├─────────────────────────────────────┤
│  Верхний колонтитул страницы        │
├─────────────────────────────────────┤
│  Заголовок группы                   │
│    Строка данных 1                  │
│    Строка данных 2                  │
│  Примечание группы (итоги)          │
├─────────────────────────────────────┤
│  Нижний колонтитул страницы         │
├─────────────────────────────────────┤
│          ПРИМЕЧАНИЕ ОТЧЁТА          │
└─────────────────────────────────────┘
```

## 5. Типы данных в MS Access

При создании таблицы каждому полю необходимо задать тип данных. Тип данных определяет, какую информацию можно хранить в поле и сколько места она занимает.

| Тип данных | Описание | Пример | Размер |
|-----------|----------|--------|--------|
| **Текстовый (Short Text)** | Строки до 255 символов | Фамилия, Город | До 255 байт |
| **Длинный текст (Long Text)** | Текст до 65 535 символов | Описание, Примечание | До 1 ГБ |
| **Числовой (Number)** | Целые и дробные числа | Возраст, Количество | 1–8 байт |
| **Дата/время (Date/Time)** | Даты и время | Дата рождения | 8 байт |
| **Денежный (Currency)** | Денежные суммы | Цена, Зарплата | 8 байт |
| **Счётчик (AutoNumber)** | Автоматически увеличивается | ID записи | 4 байта |
| **Логический (Yes/No)** | Истина или ложь | Оплачено, Активен | 1 бит |
| **Объект OLE** | Файлы, изображения | Фото, документ | До 1 ГБ |
| **Гиперссылка** | Ссылки на веб-страницы | URL сайта | До 8192 символов |
| **Вложение (Attachment)** | Прикреплённые файлы | Резюме, скан | До 2 ГБ |
| **Вычисляемый (Calculated)** | Результат формулы | Итого = Цена × Кол-во | Зависит от формулы |

## 6. Ключевое поле

**Ключевое поле (Primary Key / Первичный ключ)** — это поле (или набор полей) в таблице, значения которого **уникально идентифицируют** каждую запись.

### Требования к ключевому полю:

- 🔑 **Уникальность** — нет двух записей с одинаковым значением ключа
- 🔑 **Обязательность** — поле не может быть пустым (NULL)
- 🔑 **Неизменность** — значение ключа не должно меняться со временем

### Зачем нужен первичный ключ?

```
Без ключа:                    С ключом:
┌──────────┬──────────┐       ┌────┬──────────┬──────────┐
│ Фамилия  │ Телефон  │       │ ID │ Фамилия  │ Телефон  │
├──────────┼──────────┤       ├────┼──────────┼──────────┤
│ Иванов   │ 555-1234 │       │  1 │ Иванов   │ 555-1234 │
│ Иванов   │ 555-9876 │  →    │  2 │ Иванов   │ 555-9876 │
│ Петров   │ 555-1234 │       │  3 │ Петров   │ 555-1234 │
└──────────┴──────────┘       └────┴──────────┴──────────┘
Неясно, какой Иванов?         Каждая запись — уникальна!
```

### Типы ключей в Access:

- **Простой ключ** — одно поле (обычно поле-счётчик `ID`)
- **Составной ключ** — несколько полей вместе образуют уникальную комбинацию
- **Внешний ключ (Foreign Key)** — поле, ссылающееся на первичный ключ другой таблицы

## 7. Связи между таблицами

**Связь между таблицами** — это логическое соединение двух таблиц через общее поле. Связи позволяют объединять данные из разных таблиц и избегать дублирования информации.

### Зачем нужны связи?

Без связей пришлось бы дублировать данные:

```
❌ Плохо (без связей — данные дублируются):
┌──────────────────────────────────────────────────────┐
│ Заказ │ Клиент  │ Телефон клиента │ Адрес клиента    │
├───────┼─────────┼─────────────────┼──────────────────┤
│  001  │ Иванов  │   +996 555-0001 │ ул. Ленина, 5    │
│  002  │ Иванов  │   +996 555-0001 │ ул. Ленина, 5    │  ← дублирование!
│  003  │ Петров  │   +996 555-0002 │ пр. Манаса, 12   │
└───────┴─────────┴─────────────────┴──────────────────┘

✅ Хорошо (со связями — данные не дублируются):
Таблица "Клиенты"          Таблица "Заказы"
┌────┬─────────┬──────────┐  ┌───────┬───────────┐
│ ID │ Клиент  │ Телефон  │  │ Заказ │ КлиентID  │
├────┼─────────┼──────────┤  ├───────┼───────────┤
│  1 │ Иванов  │ 555-0001 │◄─┤  001  │     1     │
│  2 │ Петров  │ 555-0002 │  │  002  │     1     │
└────┴─────────┴──────────┘  │  003  │     2     │
                              └───────┴───────────┘
```

### Целостность данных:

MS Access поддерживает **ссылочную целостность** — систему правил, которая гарантирует, что связи между записями остаются корректными (нельзя удалить клиента, пока у него есть заказы).

## 8. Типы связей

### 🔗 Один-ко-многим (One-to-Many) — самый распространённый тип

**Описание:** Одна запись в главной таблице соответствует **множеству** записей в подчинённой таблице.

```
Таблица "Преподаватели"     Таблица "Предметы"
┌────┬──────────────┐        ┌────┬────────────────┬──────────┐
│ ID │ ФИО          │        │ ID │ Предмет        │ ПрепID   │
├────┼──────────────┤        ├────┼────────────────┼──────────┤
│  1 │ Иванов А.И.  │───┐    │  1 │ Математика     │    1     │
│  2 │ Петрова М.С. │   ├───►│  2 │ Физика         │    1     │
└────┴──────────────┘   │   │  3 │ Информатика    │    1     │
                         │   │  4 │ Химия          │    2     │
                         └───┤  5 │ Биология       │    2     │
                             └────┴────────────────┴──────────┘
Один преподаватель → много предметов
```

**Примеры:**
- Один отдел → много сотрудников
- Один клиент → много заказов
- Одна группа → много студентов

---

### 🔗 Один-к-одному (One-to-One)

**Описание:** Одна запись в таблице А соответствует **ровно одной** записи в таблице Б, и наоборот.

```
Таблица "Сотрудники"        Таблица "Паспортные данные"
┌────┬─────────┐              ┌────┬───────────────┬─────────┐
│ ID │ ФИО     │              │ ID │ Паспорт №     │ СотрID  │
├────┼─────────┤              ├────┼───────────────┼─────────┤
│  1 │ Иванов  │◄────────────►│  1 │ AN 1234567    │    1    │
│  2 │ Петров  │◄────────────►│  2 │ AN 7654321    │    2    │
└────┴─────────┘              └────┴───────────────┴─────────┘
Один сотрудник ↔ один паспорт
```

**Применяется для:**
- Разделения таблицы с секретными данными
- Хранения редко используемых полей отдельно
- Расширения структуры без изменения основной таблицы

---

### 🔗 Многие-ко-многим (Many-to-Many)

**Описание:** Множество записей таблицы А связано с множеством записей таблицы Б. Реализуется через **промежуточную (связующую) таблицу**.

```
Таблица "Студенты"    Связующая таблица       Таблица "Курсы"
┌────┬────────┐        ┌─────────┬─────────┐   ┌────┬───────────────┐
│ ID │ ФИО    │        │ СтудID  │ КурсID  │   │ ID │ Название      │
├────┼────────┤        ├─────────┼─────────┤   ├────┼───────────────┤
│  1 │ Алия   │───────►│    1    │    1    │◄──┤  1 │ Python        │
│  2 │ Бекзод │   ┌───►│    1    │    2    │◄┐ │  2 │ Web-дизайн    │
│  3 │ Гүлзар │   │    │    2    │    1    │ └─┤  3 │ Базы данных   │
└────┴────────┘   │    │    2    │    3    │◄──┤    │               │
                  │    │    3    │    2    │   └────┴───────────────┘
                  └────┤    3    │    3    │
                       └─────────┴─────────┘
Студент может учиться на нескольких курсах.
Курс может посещаться несколькими студентами.
```

**Примеры:**
- Студенты и курсы (студент записан на много курсов, курс посещают много студентов)
- Книги и авторы (у книги может быть несколько авторов, автор пишет несколько книг)
- Товары и заказы (в заказе много товаров, один товар — во многих заказах)

## 10. Практическое применение MS Access

MS Access широко используется в малом и среднем бизнесе, образовании и государственных структурах.

### 🏢 Бизнес и торговля
- **Учёт товаров** — каталог продукции, остатки на складе
- **Клиентская база** — контакты, история покупок, сегментация
- **Управление заказами** — от оформления до доставки
- **Бухгалтерия** — первичный учёт доходов и расходов

### 🏥 Медицина
- Регистратура: картотека пациентов
- Учёт медикаментов в аптеке
- Расписание приёмов врачей
- База историй болезней

### 🎓 Образование
- Журнал успеваемости студентов
- Расписание занятий
- Учёт библиотечного фонда
- База данных абитуриентов

### 🏛️ Государственные структуры
- Реестры населения
- Учёт имущества
- Архивы документов
- Кадровый учёт

### 💻 IT и разработка
- Прототипирование баз данных
- Внутренние корпоративные инструменты
- Малые веб-приложения (через SharePoint)

```
MS Access = СУБД + Таблицы + Запросы + Формы + Отчёты
           ↓
   Полноценная реляционная база данных
           ↓
   Для малого/среднего бизнеса и образования
```

**MS Access** — мощный и доступный инструмент для тех, кто хочет организовать работу с данными без глубоких технических знаний. Он объединяет в себе хранилище данных, инструменты для их обработки и удобный пользовательский интерфейс — всё в одной программе.
